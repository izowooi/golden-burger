폴리마켓 시장 예측 자동 봇 명세서
자동으로 베팅해서 yes 혹은 no 를 구매할 수 있어야한다.
구매하는 조건은 yes 혹은 no 가 ( 이하 예측 판단 ) 80% 이상 일때이다.
또한 구매 조건에는 유동성 확보 차원에서 규모 또한 포함된다. 거래 규모가 너무 적은 경우에는 들어가지 않는다. ( 이 또한 조절 가능 )
이때 베팅하는 조건의 퍼센트지와 금액은 코드가 아닌 데이터로 떼어내서 손쉽게 조절할 수 있어야한다.
80% 일 때 구매한 예측 판단은 90% 일 때 매도 처리하여 확정적인 10% 남짓의 수익을 목표로한다.
이때 90% 의 매도 조건이 되는 수치 또한 손쉽게 조절할 수 있어야한다.
또한 매도는 거래 규모를 따지지 않고, 들어간다.
인증과 관련된 부분은 API 를 사용하며 이는 환경변수에 등록해서 사용한다.
파이썬의 경우 dotenv 를 통해 사용할 수 있다.
예측하는 시장의 범위는 스포츠가 아닌 분야로 한정한다. 스포츠의 경우 랜덤이 너무 심하다.
이상의 알고리즘은 정보의 확산속도로 인해 80% 를 돌파한 예측은 90%가 언젠가 도달한다는 가정에 의하며, 이 가정이 잘못된 경우는 예상하지 않는다.
만일 가정에 잘못이 있다면 숫자의 범위를 조절해서 예측의 범위를 높인다. 그렇기 위해 80, 90 의 숫자를 편하게 조절해야한다.
또한 이 알고리즘은 시간이 지날 수록 사건의 확률이 실제 이벤트의 확률로 수렴할 것이라는 전제로 작성된다.
만일 확률이 계단식으로 급변해서 80% 를 (첫 번째 매수 시점) 생략하고 바로 90% 이상으로 (두 번째 매도 시점) 치솟을 경우는 매수/매도 아무런 행위도 하지 않는다.
한 번 이렇게 이루어진 거래는 다시 확률이 떨어져서 80% 를 지나도 추가로 매수하지 않는다.
만일 확률이 80% 을 넘었으나 다시 떨어지는 경우는 그대로 둔다. ( 손절 타이밍은 없다 )
만일 확률이 80% 넘기고, 90% 달성 못해도 그대로 둔다.
만일 확률이 80% 을 넘기고, 90% 넘기고, 다시 아래로 내려가서 80을 다시 넘긴다고 해도 추가로 거래하지 않는다. ( 한 번 거래한 시장은 다시 거래하지 않는다 )

DB 의 경우 불필요한 네트워크 비용을 아끼기 위해 로컬에서 파일로 보관한다.
DB 의 저장된 데이터를 통해 현재 어떤 거래에 대해서 오더북을 갖고 있으며, 언제 샀으며 언제 청산했는 지 등을 알아야한다.
DB 에 저장된 데이터를 통해 특정 날짜에 몇건의 거래를 수행했는 지도 알 수 있어야한다.
이러한 데이터를 통해 매수시점과 청산시점의 수치, 매수 시점의 거래규모, 매수 하는 가격등을 결정 짓는다.

본 봇은 젠킨스에서 스케쥴러에 의해 자동으로 실행되는 것을 전재로한다.
이때 실행되는 주기는 최대한 짧게 가져가는 것을 목표로 하되, polymarket 1일 한도, 일주일 한도 등 한도에 걸리지 않아야한다.
또한 젠킨스에서 수치를 다르게 해서 서로 다른 잡을 실행하여야 하기 때문에 DB 는 각각의 잡의 하위 폴더에 갖고 있어야한다.

리드미에는 API 키 생성하는 방법을 포함해서 상세한 문서를 담아야한다.
리드미는 mermaid 등의 시각화 자료를 통해 보다 쉽게 알 수 있도록 도움을 주어야한다.

API 키를 테스트할 수 있는 스크립트가 포함되어야한다.
