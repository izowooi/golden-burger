현재의 전략은 단순히 거래규모가 100000 달러이면서 80~90% 인 거래를 매수하고,
별다른 손절 조건없이 오로지 90% 을 넘길 때 매도하는 게 전략의 다입니다.
이는 여러가지 문제가 있다고 보여지며 이 부분은 polymarket-strategy-claude.md 에서도 동일하게 지적합니다.
이에 개선안을 받아들여 아래와 3가지 기능을 넣어서 전략을 변경하고자 합니다.
1. 확률 구간 조정 ( 85~97% 구간 ), 거래규모 변경 ( 50000 )
2. 모멘텀 기반으로 변경
2. 손절 조건 추가
먼저 확률 구간입니다. 기존의 80~90 구간보다 85~97 구간이 더 우월하다고 합니다.
그러므로 기본적으로 인자로 받지 않으면 85% ~ 97% 구간으로 변경해주세요.
또한 기본 거래 규모도 100000 에서 50000 으로 변경합니다.
두번째는 가장 핵심이 되는 정적 임계값에서 동적 모멘턴으로 조건을 변경하는 것입니다.
저는 5분에 한 번씩 요청을 보내기 때문에 최근 15분간의 모멘텀과 6시간의 모멘텀을 계산할 수 있습니다.
단순히 수동적으로 기다리지 말고, 상승 모멘텀이 확인되면 진입하고, 이후에 모멘텀이 약화되거나 지정된 수익을 ( 10% ) 달성하면 매도합니다.
마지막으로 손절 조건을 추가하여 진입가 대비 -10% 가 되면 손절합니다.
이를 정리하면 다음과 같습니다.

[진입 조건]
- 확률 구간: 85% ≤ p ≤ 97% (기존 80-90%에서 상향)
- 진입 필터: 15분간의 모멘텀이 6시간의 모멘텀에 비해 의미있게 앞지르는 경우 ( golden cross )

[청산 조건]
- 이익실현: 확률 97%+ 또는 진입가 대비 +7%
- 모멘텀 청산: 6시간의 모멘텀에 비해 15분간의 모멘텀이 의미있게 뒤쳐지는 경우 ( dead cross )

[손절]
- 진입가 대비 -10% (기존 손절 없음에서 변경)
