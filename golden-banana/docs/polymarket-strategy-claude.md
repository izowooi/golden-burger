# 폴리마켓 자동매매 봇 전략 최적화: 학술 연구와 실증 데이터 기반 종합 분석

예측 시장에서 **80-90% 확률 구간의 매수 전략은 학술적으로 유효**하지만, 현재 전략에는 세 가지 핵심 개선점이 존재한다. 첫째, 80% 진입 임계값은 행동경제학적 편향을 활용하기에 약간 이른 수준이며 **85-95% 구간이 더 효율적**이다. 둘째, 고정 임계값 대신 **모멘텀 기반 동적 진입**이 더 높은 수익률을 제공한다. 셋째, $100K 볼륨 필터는 적절하나 **시장 카테고리별 차별화**가 필요하다. 이 연구는 Page & Clemen(2013), Snowberg & Wolfers(2010), 그리고 최근 Polymarket 1.24억 거래 분석 데이터를 기반으로 구체적 전략 개선안을 제시한다.

## 예측 시장 캘리브레이션: 80%는 실제로 몇 %인가

예측 시장의 가격이 실제 발생 확률과 얼마나 일치하는지는 전략 수립의 핵심이다. Polymarket의 Brier Score는 **0.058**(12시간 전 예측 기준)로, 이는 기상 예보(0.05~0.12)와 유사한 매우 정확한 수준이다.

그러나 Polymarket 캘리브레이션 분석에 따르면, **표시 확률이 실제 발생률보다 약간 높게** 나타나는 경향이 있다. 80%로 표시된 이벤트는 실제로 **77-79%** 확률로 발생한다. 이 2-4%포인트의 과대추정은 일관되게 관찰되며, 주요 원인으로 공매도의 어려움, AMM에서 CLOB으로의 구조 변화, 그리고 심리적 "Yes 선호 편향"이 지목된다.

**Favorite-Longshot Bias**는 예측 시장에서 50년 이상 지속된 현상이다. Snowberg & Wolfers(2010)의 NBER 연구에 따르면:

| 확률 구간 | 시장 효율성 | 편향 방향 |
|-----------|-------------|-----------|
| 0-15% | **비효율적** | 과대평가 (실제 발생률 더 낮음) |
| 15-85% | 상대적 효율적 | 균형 |
| 85-95% | 약간 비효율적 | **과소평가** (실제 발생률 더 높음) |
| 95%+ | 비효율적 | **상당히 과소평가** |

Reichenbach & Walther(2024)의 **1.24억 건 Polymarket 거래 분석** 결과는 주목할 만하다. 일반적인 예측 시장과 달리 Polymarket에서는 **전반적인 longshot bias가 관찰되지 않았다**. 시장 가격은 실제 발생 확률을 상당히 정확하게 추적했으며, 오히려 북메이커 오즈보다 약간 뛰어났다. 그러나 **계약 초기 단계와 만기 직전**에 부정확성이 집중되는 패턴이 발견되었다.

## 행동경제학적 편향이 만드는 가격 비효율성

Kahneman과 Tversky의 전망이론(Prospect Theory)은 예측 시장의 가격 형성 메커니즘을 설명하는 핵심 프레임워크다. 확률가중함수(Probability Weighting Function)에 따르면, 사람들은 확률을 객관적 수치 그대로 인식하지 않는다.

**확실성 효과(Certainty Effect)**는 95%+ 구간에서 가장 강하게 작용한다. Tversky & Kahneman(1992) 연구에 따르면, **95% 확률에 대한 심리적 가중치는 약 80%**에 불과하다. 즉, 사람들은 "거의 확실한" 이벤트를 심리적으로 15%포인트나 할인한다. 이는 95% 이상 이벤트가 **체계적으로 과소평가**되어 매수 기회를 제공함을 의미한다.

| 확률 구간 | 심리적 가중 | 시장 영향 | 거래 전략 |
|-----------|-------------|-----------|-----------|
| 0-5% | 과대평가 (π(p) > p) | **고평가** | 매도/회피 |
| 5-15% | 상당히 과대평가 | **고평가** | 매도 |
| 15-85% | 거의 선형 | 적정 가격 | 중립 |
| 85-95% | 과소평가 | **저평가** | 매수 검토 |
| 95-100% | 상당히 과소평가 | **저평가** | 적극 매수 |

**손실회피(Loss Aversion)**도 중요한 역할을 한다. λ ≈ 2.25(손실의 고통이 이익의 기쁨보다 2.25배)로 추정되며, 이는 트레이더들이 포지션 진입가를 기준점으로 삼아 승자는 너무 오래 보유하고 패자는 너무 빨리 매도하는 경향을 만든다.

## 모멘텀 전략: 예측 시장에서 실제로 작동하는가

**562개 Polymarket 선거 시장**(2020-2024, $50K+ 거래량) 분석 결과, 모멘텀은 존재하지만 조건이 있다.

**움직임 크기에 따른 모멘텀 강도**:
- **2-3% 일일 변화**: 같은 방향으로 추가 이동할 확률 **6-8% 상승** (가장 강한 모멘텀)
- **5% 일일 변화**: 추가 이동 확률 **2-3% 상승** (중간 강도)
- **10%+ 일일 변화**: 모멘텀 **약화**, 평균회귀 가능성 높음

핵심 발견은 **대형 움직임(10%+)은 모멘텀과 상관관계가 낮다**는 점이다. 이러한 급격한 가격 변동은 이익실현과 역추세 베팅을 유발하여 오히려 반전하는 경향이 있다.

**시간대별 최적 전략**:

| 시간대 | 지배적 패턴 | 권장 전략 |
|--------|-------------|-----------|
| 분~시간 | 평균회귀 | 극단적 움직임에 역추세 |
| 1-3일 | 모멘텀 | 소/중형 움직임 추세추종 |
| 만기 직전 | 정보 기반 | 이벤트 분석 |

**최적 룩백 기간**:
- 모멘텀 시그널 감지: **24시간**
- 평균회귀 진입: **1-4시간**
- 트렌드 확인: **7일 이상**

추천하는 기술적 지표 조합은 **EMA(20)** + **SMA(50)** + **ROC(12)**이다. ROC(12)가 ±10%를 초과하면 과매수/과매도 신호로 해석한다.

## 통계적으로 유의미한 임계값 설정 방법

현재 80%/90% 임계값이 직관적으로 설정되었다는 문제를 해결하기 위한 정량적 프레임워크를 제시한다.

### Kelly Criterion 적용

예측 시장용 Kelly 공식:

```
f* = (p - pm) / (1 - pm)

p = 추정 실제 확률
pm = 시장 가격 (현재 확률)
f* = 최적 배팅 비율
```

**예시**: 실제 확률 88%로 추정, 시장 가격 82%
- Edge = 0.88 - 0.82 = 0.06 (6%)
- Kelly = 0.06 / 0.18 = 33.3%

**중요**: Full Kelly는 절대 사용하지 말 것. Wharton 연구(Beggy et al., 2023)에 따르면:
- Full Kelly 사용 시 **100% 파산** 시나리오
- **Half Kelly(50%)** + 10% 마진 임계값이 가장 수익성 높음
- **Quarter Kelly(25%)**가 불확실성이 높을 때 권장

| Kelly 비율 | 사용 조건 | Full Kelly 대비 성장률 |
|------------|-----------|----------------------|
| 100% | 완벽한 확률 지식 | 100% (그러나 파산 위험) |
| 50% | 중간 불확실성 | ~75% |
| 25% | **권장 기본값** | ~56% |
| 12.5% | 매우 높은 불확실성 | ~37% |

### 임계값 선택 휴리스틱

```
최소 필요 Edge = 2 × 거래비용 + 안전마진

예: 거래비용 1%, 안전마진 5%
최소 Edge = 2(0.01) + 0.05 = 7%

시장 가격 80%에서 거래하려면:
추정 실제 확률 ≥ 80% + 7% = 87% 이어야 함
```

### 통계적 유의성 확보

| 샘플 크기 | 95% 신뢰구간 폭 | 임계값 결정 적합성 |
|-----------|-----------------|-------------------|
| 50개 | ±14% | 불충분 |
| 200개 | ±7% | 합리적 |
| 1,000개 | ±3% | 양호 |
| **2,000개+** | ±2% | **권장** |

## $100K 볼륨 필터의 타당성과 재평가

**핵심 발견**: 거래량이 높다고 반드시 더 정확한 것은 아니다.

Vanderbilt 대학 연구(Clinton & Huang, 2024-2026)의 충격적인 결과:

| 플랫폼 | 정확도 | 거래량 특성 |
|--------|--------|-------------|
| PredictIt | **93%** | 포지션 제한 ($850), 소규모 |
| Kalshi | 78% | 중간 규모 |
| Polymarket | 67% | **$2B+ 최대 규모** |

**"Volume Trap" 현상**: 고볼륨 플랫폼은 정보에 기반한 트레이딩보다 투기적 노이즈와 당파적 "응원"을 유인한다. 대형 "고래" 포지션($30M+)은 "서사적 중력"을 만들어 소규모 트레이더들이 분석 대신 추종하게 만든다.

**카테고리별 시장 효율성** (Kalshi 7,210만 거래 분석):

| 카테고리 | Maker-Taker Gap | 효율성 |
|----------|-----------------|--------|
| 금융 | 0.17pp | **거의 완벽** |
| 정치 | 1.02pp | 중간 |
| 스포츠 | 2.23pp | 중하 |
| 암호화폐 | 2.69pp | 낮음 |
| 엔터테인먼트 | 4.79pp | 매우 낮음 |

**권장 볼륨 임계값**:

| 용도 | 최소 볼륨 | 최소 유동성 |
|------|-----------|-------------|
| 가격 신호 신뢰성 | $10K+ | $5K+ |
| 소형 포지션 ($100-$1K) | $50K+ | $10K+ |
| 중형 포지션 ($1K-$10K) | **$100K+** | $25K+ |
| 대형 포지션 ($10K+) | $500K+ | $100K+ |

**슬리피지 추정**:

| 시장 볼륨 | 예상 슬리피지 |
|-----------|---------------|
| <$10K | 5-15%+ (시장가 금지) |
| $50K-$250K | 0.5-2% |
| >$1M | <0.2% |

## Polymarket 기술 구조와 API

### 시장 구조: CLOB (중앙 지정가 주문장)

Polymarket은 **AMM이 아닌 CLOB**를 사용한다. 2022년 말 AMM에서 CLOB으로 전환했다.

- **하이브리드-분산형**: 오프체인 주문 매칭 + 온체인 결제
- **결제**: Polygon 네트워크, USDC.e 담보
- **토큰 표준**: Gnosis CTF (ERC1155)
- **오라클**: UMA Optimistic Oracle

### API 구조

**CLOB API** (거래): `https://clob.polymarket.com`
- `GET /price` - 현재 가격
- `GET /book` - 오더북 스냅샷
- `POST /order` - 주문 실행
- 배치 주문: 최대 15개/배치

**WebSocket**: `wss://ws-subscriptions-clob.polymarket.com/ws/market`
- 실시간 Level 2 가격 데이터
- 5분 폴링보다 WebSocket 권장

**Gamma API** (메타데이터): `https://gamma-api.polymarket.com`
- 시장 목록, 카테고리, 해결 소스

### 수수료 구조

| 시장 유형 | Maker | Taker |
|-----------|-------|-------|
| 일반 시장 | **0%** | **0%** |
| 15분 암호화폐 | 0% | 최대 3.15% |

**일반 시장에서는 거래 수수료가 없다**는 점이 중요하다.

## 전략 개선 권장사항

### 현재 전략의 문제점 해결

**문제 1: 조건 만족 종목 6개뿐**
- 해결: 볼륨 기준을 $50K로 낮추되, 카테고리 필터 추가 (금융 > 정치 > 스포츠)
- 확률 구간을 **85-97%**로 확장 (행동경제학적 비효율성 최대 구간)

**문제 2: 80%→90% 대기 시간 과다**
- 해결: 모멘텀 기반 동적 청산 조건 추가
- 24시간 내 2-3% 상승 시 추세추종, 10%+ 급등 시 즉시 이익실현

**문제 3: 임계값의 통계적 근거 부재**
- 해결: 아래 개선된 파라미터 사용

### 개선된 전략 파라미터

```
[진입 조건]
- 확률 구간: 85% ≤ p ≤ 97% (기존 80-90%에서 상향)
- 최소 볼륨: $50K (카테고리별 차등)
- 진입 필터: 24시간 내 2-3% 상승 모멘텀 확인
- Edge 요구: 최소 5% (추정 실제확률 - 시장가격)

[청산 조건]  
- 이익실현: 확률 97%+ 또는 진입가 대비 +7%
- 모멘텀 청산: 24시간 내 5%+ 하락 시
- 시간 청산: 만기 24시간 전

[손절]
- 진입가 대비 -10% (기존 손절 없음에서 변경)
- Kelly 기반 포지션 사이징: 25% Kelly

[포지션 사이징]
Kelly_fraction = 0.25
Position_size = Kelly_fraction × ((estimated_prob - market_prob) / (1 - market_prob)) × 총자본
```

### 하이브리드 전략 제안

**전략 A - 모멘텀 추종** (소/중형 움직임):
- 진입: 24시간 내 2-3% 상승, EMA(20) 위
- 보유: 1-3일
- 청산: 모멘텀 약화 또는 +5% 수익

**전략 B - 평균회귀** (대형 움직임):
- 진입: 24시간 내 10%+ 급등 후 역추세
- 조건: 60%+ 종목이 하락했거나, 40% 미만 종목이 급등
- 청산: 가격이 50% 회복 또는 48시간 내

## 결론: 핵심 인사이트와 실행 계획

학술 연구와 실증 데이터를 종합하면 세 가지 핵심 통찰이 도출된다.

첫째, **85-97% 확률 구간이 80-90%보다 우월**하다. 확실성 효과로 인해 95%+ 이벤트는 심리적으로 15%포인트 할인되어 체계적 저평가가 발생한다. 이 구간에서 "거의 확실한" 이벤트를 매수하는 것이 행동경제학적 비효율성을 최대로 활용하는 방법이다.

둘째, **정적 임계값보다 동적 모멘텀 조건이 효과적**이다. 80%에서 90%까지 수동적으로 기다리는 대신, 24시간 내 2-3% 상승 모멘텀이 확인되면 진입하고, 10%+ 급등 시 평균회귀를 예상하여 이익실현하는 전략이 더 높은 위험조정수익률을 제공한다.

셋째, **볼륨은 정확성의 충분조건이 아니다**. PredictIt(93%)이 Polymarket(67%)보다 정확했다는 Vanderbilt 연구 결과는 "더 많은 거래량 = 더 정확한 가격"이라는 직관에 반한다. 다양한 참여자 구성이 대규모 단일 방향 자금보다 중요하다.

실행 계획으로는 먼저 Polymarket CLOB API를 통해 과거 6개월 데이터로 백테스팅을 실시하고, Monte Carlo 시뮬레이션(10,000회 이상)으로 위험 수준을 검증한 후, $1,000 규모로 파일럿 운영하여 실제 슬리피지와 실행 품질을 측정하는 것을 권장한다.
